# Architecture Overview

This project follows a layered runtime pipeline:

`RuntimeRunner -> Strategy -> Intent -> Motor Controller -> Executor`

## Layer Responsibilities

### RuntimeRunner
- Orchestrates the end-to-end runtime lifecycle.
- Coordinates planning, execution flow, and runtime state transitions.
- Delegates decision-making and execution to downstream layers.

### Strategy
- Selects the current behavioral approach based on context and goals.
- Chooses which intent path to activate.
- Stays focused on policy and high-level behavior, not low-level execution details.

### Intent
- Defines **what** action should be performed next.
- Converts strategic goals into explicit, domain-level intents.
- Must remain independent from low-level motor mechanics.

### Motor Controller
- Defines **how** an intent is physically executed.
- Handles timing, movement primitives, and interaction realism.
- Translates intent-level actions into executor-ready command sequences.

### Executor
- Performs concrete command execution against runtime targets.
- Applies the final operational commands generated by the motor layer.
- Focuses on reliable command dispatch and completion reporting.

## Boundary Rules
- Intent and Motor Controller are distinct layers and must not be merged.
- Upstream layers decide objectives; downstream layers perform execution details.
- Changes should preserve this separation to prevent behavioral drift and accidental architectural collapse.
